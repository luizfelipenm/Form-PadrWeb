3) Na tag < input > para que serve o atributo "value"?

O atributo value em HTML é usado para especificar o valor do elemento com o qual é usado. Tem um significado diferente para diferentes elementos HTML. O atributo de valor pode ser usado para definir um valor padrão para um campo de entrada, definir valores para botões de opção, caixa de seleção, elementos de opção e definir texto de exibição de um botão

5) Para que serve a tag < label >? Para que serve o atributo "for" nessa tag?

O atributo for faz um tipo de link entre a label e o input. Logo o for tem que apontar para o ID do input. Outra forma de uso é que vc pode fazer vários tipos de customização pelo CSS escondendo o input default do browser por exemplo e usando a própria label como se fosse o botão. Representa uma legenda para um item em uma interface de usuário. O atributo for serve para referenciar o id da tag input, deve conter o mesmo nome.

6) Na tag < option > para que serve o atributo "selected"?

Em um formulário Web, o elemento HTML option é usado para criar um controle que representa um item dentro de um elemento HTML5 select, optgroup ou datalist.

7) Nas tags de formulário para que serve o atributo "required"?

Colocar o atributo required funciona para que o usuário não envie o formulário sem algum campo preenchido. Ao estuda sobre back-end, e sobre segurança do banco de dados eu descobri que existe um risco de apenas o usar o required no HTML. O usuário pode remover esse atributo através do console do navegador.

8) Na tag < form > para que serve o atributo "action"?

O atributo action define o local (uma URL) em que os dados recolhidos do formulário devem ser enviados. O atributo method define qual o método HTTP para enviar os dados (ele pode ser "GET" ou "POST" (veja as diferenças aqui). Para que utilizamos a tag form?

9) Na tag < form > para que serve o atributo "method"?

O atributo method define o método HTTP com que o formulário HTML irá lidar com os dados recebidos. São eles: o método GET e o método POST.
